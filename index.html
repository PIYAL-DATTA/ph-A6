Frequently Asked Questions

1. The difference between var, let, and const
'var' is function-scoped and hoisted (can be redeclared and reassigned)
'let' is block-scoped (can be reassigned but not redeclared)
'const' is block-scoped (cannot be reassigned or redeclared).
2. The difference between map(), forEach(), and filter()
map(): Transforms each item in an array and returns a new array with the transformed values.
forEach(): Executes a function on each array item but does not return anything.
filter(): Returns a new array containing only the items that pass a condition.
3. Explain arrow functions and how they are different from regular functions
Arrow functions in JavaScript are a concise way to write functions using the `=>` syntax, introduced in ES6. Unlike regular functions, arrow functions do not have their own `this` context; instead, they inherit `this` from their enclosing scope, making them useful for callbacks and situations where you want to preserve the outer `this`. They also cannot be used as constructors and do not have access to the `arguments` object. While arrow functions offer a cleaner, shorter syntax, regular functions are more suitable when you need dynamic `this` binding or are creating object methods or constructors.
4. How JavaScript Promises work?
JavaScript Promises represent the eventual completion (or failure) of an asynchronous operation and its resulting value. A Promise is an object that can be in one of three states: **pending**, **fulfilled**, or **rejected**. When a Promise is created, it starts in the pending state and moves to fulfilled with a value when the operation succeeds, or to rejected with a reason (error) if it fails. You handle the result using `.then()` for success, `.catch()` for errors, and optionally `.finally()` for cleanup. Promises help write cleaner, more manageable async code compared to deeply nested callbacks (callback hell).
5. How closures work in JavaScript?
Closures in JavaScript occur when a function **remembers and accesses variables from its lexical (outer) scope**, even after that outer function has finished executing. This means an inner function retains access to the variables of its parent function, allowing those variables to persist in memory. Closures are created every time a function is defined inside another function and used outside its scope. They are commonly used for data privacy, creating function factories, and maintaining state across function calls without using global variables.
